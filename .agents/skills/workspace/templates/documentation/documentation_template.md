# Documentation Template

## Purpose

This document defines the standard format for engineering documentation generated within this workspace.

Whenever documentation is created, follow this document unless explicitly instructed otherwise.

---

# Documentation Rules

Use Markdown format.

Keep the document clear, concise and professional.

Use simple English.

Use meaningful headings.

Use relative paths when referring to files within the workspace.

Do not include unnecessary explanations or duplicated information.

---

# Standard Document Structure

Use the following structure whenever applicable.

```text
# Title

## Purpose

## Scope

## Prerequisites

## Files Used

## Procedure

## Results

## Verification

## References

## Revision History
```

Omit sections that are not applicable.

Do not create additional sections unless required.

---

# Revision History

Maintain a revision history whenever a document is updated.

Use the following format.

| Version | Date | Description |
|---------|------|-------------|

---

# General Rules

Use consistent terminology throughout the document.

Keep formatting consistent across all project documents.

Do not invent technical information.

If required information is unavailable, ask the user before generating the document.

Follow this template unless the user specifies a different document format.
